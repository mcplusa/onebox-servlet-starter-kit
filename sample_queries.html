Sample Directory OneBox Module Usage

Check out EmployeeDirectory.java for the entire directory of people that you can query for in the sample organization.

If you are running the default sample OneBox provider (SampleNoAuthOneBoxProvider.java) which requires no authorization, then you should be able to query the directory and get detailed contact information for anyone in the organization. Here is a sample search to exercise your OneBox.

Search term Expect to see
directory Brown This will query the directory and return detailed contact information for all three people in the organization whose last name is Brown.

 

If you are running one of the sample secure OneBox providers (SampleBasicAuthOneBoxProvider.java, SampleLDAPAuthOneBoxProvider.java, or SampleSSOAuthOneBoxProvider.java) then see the provider source file for the user ids, roles, and passwords (if applicable) for the full set of users that you can authenticate as.

Here are some sample searches to exercise the SampleBasicAuthOneBoxProvider (or the other secure providers) and see the effect of user level access control in OneBox. Assuming you are using a Google Search Appliance as your OneBox client and you are using the appliance's default user interface, you'll need to select "public and secure content" before running each search. You'll also need to clear your browser cache (or restart your browser) in between search attempts to retain the ability to log in as a different user on each attempt.

Search term Log in as Expect to see
directory Brown wbrown/wbrown Since William Brown is a contractor, he is only authorized to see the contact information on file for himself despite the existence of other Browns in his department.
directory Brown jsmith/jsmith Since James Smith is an employee in the Marketing department, he is authorized to see some basic directory information on all of the people in his department, including both Browns.
directory Brown sbrown/sbrown Since Susan Brown is an employee in the Marketing department, she is authorized to see the detailed contact information on file for herself and is able to see some basic directory information on the other people in her department, including William Brown.
directory Brown rmiller/rmiller Since Richard Miller is a manager in the Marketing department, he is authorized to see the detailed contact information on file for all of the people in his department, including both Browns.
directory Brown mhernandez/mhernandez Since Margaret Hernandez is a system admin, she is authorized to see the detailed contact information on file for all of the people in all of the departments, including all three Browns.
